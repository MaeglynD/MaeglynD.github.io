When should you use Vuetify?

Vuetify revolves around material design rules, so first things first if you don't want a material design website then you probably should look into other frameworks. Secondly you’ll need some understanding of VueJS and SSR (Vue router) as the framework was built to accustom sites using them. Okay, sweet so now that that's out the way let's dive into doc’s and see why this framework is so popular.

Jumping right into the good stuff is made even easier through vuetify’s quick start page. Select you’re skill level, choose you’re developing requirements and boom. A whole host of options will be at your disposal - from codepens with vuetify already setup to ready made Vue cli / webpack templates complete with options to introduce whatever your requirements may be (A La Carte to decrease load times, babel, eslint, typescript, test scaffolding and so on). If at any point you feel overwhelmed, stuck or uninspired, I’d encourage you to check out vuetify’s community discord, where lead developers hang out to support users of all skill levels and also share their latest projects with one another. Vuetify is also pretty popular among front end dev’s, so whatever your IDE may be, chances are there's a package to help with the syntax.

Pros

  • Semantic, easy to remember framework
  • Large, active community ready to help
  • Concise, easy to learn documentation
  • Free, ready made, editable themes
  • Good for responsive, mobile-first applications
  • Its like, really pretty bro

Cons

  • Documentation lacking on deployment / production
  • Requires prerequisite knowledge of Vue, SSR and more
  • Grid system, font size, padding and margin have room for improvement

In conclusion, if you’re versed in Vue and want to try out a fantastic material design framework then, without a doubt, vuetify is for you. Perhaps you don't know much about Vue, but still want to give it a go? Try a custom theme, download a template and have some fun with it.
