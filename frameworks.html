Sass & Kaiten

Sass is a fantastic preprocessor, popular amongst most front end developers for speeding up the entire styling process. Using it has been pretty fun, and to showcase some of sass’s features I decided to create a fake cryptocurrency startup site called ‘Kaiten’. In my eyes, the three main features of sass that stood out were; nesting capabilities, variables and mixins. Nesting with sass is a piece of cake, put the child stylings within the parent stylings and wallah. This becomes useful when you’re working with multiple stylesheets or a large amount of css for clean hierarchical code. If you’re working with a high amount of media queries, it can be easy to forget a certain parent element but on whole the benefits of nesting far outway the downsides

Variables and mixins can be thought of as a little dose of JS put into the SCSS syntax. You can probably guess what variables are, and when it comes to reusing certain styles (which I do a lot), it’s a godsend. Although replicating sass in css isn’t that hard, I’m definitely not going back to vanilla css any time soon. Back to Kaiten, I’m pretty happy with the finished site, apart from playing around with sass, I also wanted to feature the use of SVG’s (hexagons inside hexagons) as well as gradients. I think we can all agree gradients make everything better. And if you don’t believe me, check out the live site at kaitencoin.github.io

Pros

  • Speeds up stylings
  • Good documentation
  • Easy to learn & setup
  • Easy to add to tech stack

Cons

  • Not needed for small scale projects
  • Possibly slower performance
  • Possible overengineering

Materialize CSS

Working with this framework has been, overall, really fun. There are certainly many benefits to using a well known design schema like material and I think subject matter is the key takeaway here. I’d say, if you have a formal subject matter (e.g. banking, commercial software retail, etc.) then definitely consider materialize and that's not to say that informal subject matters shouldn’t, but more so don’t stick to material rules as strictly. Why? Personality can be lost in the mist of perfect spacing, perfect consistency of colors, fonts and so on.

Pros

  • Lightweight (only 29kb)
  • Good documentation
  • Uses Google’s material design
  • Extremely good for mobile development
  • Angular support

Cons

  • Deviation from Material UI voids materializes purpose
  • Not supported on older browsers
